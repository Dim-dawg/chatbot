# Product Requirements Document (PRD)

## 1. Goal
Provide a secure, local AI assistant that helps judges manage their caseload, search for information, and perform routine database actions through a unified digital command center.

## 2. Target Users
- **Primary**: Judges of the Senior Courts.
- **Secondary**: Court Clerks and Administrative Staff.

## 3. High-Level Requirements
- **Security**: Zero data leakage to the cloud.
- **Ease of Use**: Natural language interface for technical database operations.
- **Speed**: Instant access to thousands of records.
- **Accuracy**: AI must be grounded in real database facts.

## 4. Functional Requirements
- The system must display live caseload metrics (Open vs. Closed).
- The system must allow searching by name or claim number.
- The system must support conversational requests to update case status.
- The system must provide a vertical timeline of a case's procedural history.
